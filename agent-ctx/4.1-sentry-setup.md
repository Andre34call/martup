# Task 4.1 — Sentry Error Monitoring Setup

## Task: Set up Sentry error monitoring for the MartUp Next.js project

### Work Completed

1. **Created `sentry.client.config.ts`** at project root
   - Initializes Sentry with `NEXT_PUBLIC_SENTRY_DSN` environment variable
   - If DSN is not set, `Sentry.init()` is never called (silent no-op)
   - Includes session replay integration (10% session sample, 100% on-error sample)
   - Text and input masking enabled for privacy
   - Traces sample rate: 10%

2. **Created `sentry.server.config.ts`** at project root
   - Server-side Sentry initialization with DSN guard
   - Traces sample rate: 10%

3. **Created `sentry.edge.config.ts`** at project root
   - Edge runtime Sentry initialization with DSN guard
   - Traces sample rate: 10%

4. **Updated `next.config.ts`** — wrapped with `withSentryConfig`
   - Imported `withSentryConfig` from `@sentry/nextjs`
   - Wrapped the existing NextConfig object preserving all options:
     - `output: "standalone"` ✅
     - `typescript.ignoreBuildErrors: true` ✅
     - `reactStrictMode: false` ✅
   - Sentry build options: `silent: true`, `hideSourceMaps: true`

5. **Created `src/lib/sentry.ts`** — shared Sentry utilities
   - `captureException(error, extra?)` — safely capture exceptions
   - `captureMessage(message, level?, extra?)` — capture manual messages
   - `setSentryUser(user)` — set user context (id, email, name, phone, role)
   - `clearSentryUser()` — clear user scope on logout
   - `addBreadcrumb(breadcrumb)` — navigation/action tracking
   - `setTag(key, value)` — scope tagging
   - `setExtra(key, value)` — extra context
   - `isSentryReady()` — check if Sentry is configured
   - All functions are safe no-ops when DSN is not configured

6. **Updated `src/components/error-boundary.tsx`**
   - Imported `captureException` from `@/lib/sentry`
   - `componentDidCatch` now sends errors to Sentry with `componentStack` as extra context
   - Existing console.error and UI behavior preserved

7. **Updated `src/components/ecommerce/providers.tsx`**
   - Imported `setSentryUser` and `clearSentryUser` from `@/lib/sentry`
   - After Google OAuth login success: calls `setSentryUser()` with user data
   - On logout (isAuthenticated becomes false): calls `clearSentryUser()`
   - All existing behavior (cart merge, WebSocket, storage setup) preserved
